## 1. AI 에이전트 정의 — 에이전트란 무엇인가

- **목표를 향해 환경을 인지(perceive) → 추론(reason) → 행동(act)하는 루프를 매 단계 인간 승인 없이 자율 반복**하는 시스템
- 핵심 표현: *"대화하는 AI"가 아니라 "일을 끝내는 AI"* — plan → act → observe → adapt
- 단순 지시 수행을 넘어 **스스로 하위 목표 설정·계획·조정**까지 수행
- 도구(검색·코드 실행·API·문서·로봇)를 사용해 외부 세계에 실제 변화를 일으키고, 결과에 따라 자기 행동을 업데이트

> **4-AI 합치**: 4개 AI 모두 "목표 지향 + 자율 루프 + 도구 사용 + 실행 주체"로 일치 정의.

## 2. 핵심 구성 요소

**5대 요소 — 인지 · 추론 · 메모리 · 계획 · 도구(행동)**

| 요소 | 내용 |
|------|------|
| **인지** (Perception) | 텍스트·이미지·로그·센서·시스템 이벤트를 읽어 현재 상태·의도 파악 |
| **추론** (Reasoning) | LLM 기반 상황 이해·다단계 사고·원인결과 추론·전략 수립·반성적 추론 |
| **메모리** (Memory) | 단기(컨텍스트) / 장기(지식·히스토리·RAG·벡터DB) / 작업 메모리(중간 산출물). **2026년 1급 아키텍처 요소** |
| **계획** (Planning) | 목표를 태스크로 분해·실행 순서 결정. 상태기계형 또는 반복 reflection |
| **도구·행동** (Tools/Action) | API·코드 인터프리터·웹·앱·로봇 호출 + 실행 후 결과 평가(Evaluate) 루프 |

## 3. 아키텍처와 설계 패턴

**레이어 구조** — 인터페이스(사용자) → 오케스트레이션(플래너·코디네이터) → 작업 에이전트 → 인프라(도구·데이터). (Anthropic식 4레이어: 추론·오케스트레이션·메모리·도구)

| 패턴 | 내용 |
|------|------|
| **ReAct** | Reasoning ↔ Acting 교차 반복, 각 단계 로그를 다음 추론에 활용 |
| **워크플로우 패턴** | DAG·상태머신·그래프(LangGraph). Anthropic 5패턴: prompt chaining · routing · parallelization · orchestrator-workers · evaluator-optimizer |
| **계층적 오케스트레이션** | 전략 상위 에이전트 + 기능 하위 에이전트 협업 |
| **하이브리드 설계** | 자율 행동 위에 정책·템플릿·가드레일을 씌워 예측 가능성 강화 |

> **핵심 통찰**: 강력한 모델 + 나쁜 아키텍처 = 일관된 실패 / 평범한 모델 + 좋은 아키텍처 = 프로덕션급 신뢰성.

## 4. 에이전트 유형·구현 형태 분류 ★필수

### 4-A. 개념적 유형 분류

**단일 기준이 아니라 여러 축으로 교차 분류한다.**

| 분류 축 | 유형 |
|---------|------|
| **자율성 기준** | 보조형 → 반자율형 → 완전자율형 / (코파일럿 → 코워커 → 오토파일럿) |
| **역할·기능 기준** | 정보검색형 · 실행형 · 의사결정형 · 협업·조율형 · 코딩형 |
| **구조 기준 (고전 AI)** | 단순 반사형 · 모델 기반 · 목표 기반 · 유틸리티 기반 · 학습 에이전트 |
| **에이전트 수** | 단일 에이전트 ↔ 멀티 에이전트 시스템(MAS) |
| **범위 기준** | 범용 오케스트레이터(총괄) · 도메인 특화(법률·의료·CS) · 도구 특화(API·앱·로봇) |
| **조직 관점** | 개인 에이전트(사용자 단위) ↔ 팀·조직 에이전트(프로세스 단위) |
| **2026 4종(species)** | 반응형(reactive) · 계획형(planning) · 협업형(collaborative) · 지속형 디지털 직원(persistent) |
| **엔터프라이즈 6유형** | Customer · Employee · Creative · Code · Data · Security |

> 실제 제품은 대개 **여러 분류가 섞인 복합형**. llm-dependent-agent-create는 Phase 1 스무고개에서 이 축들로 대상 에이전트를 좌표화한다.

**형태 선택 기준**: 누가 호출하나(사람·이벤트·타 시스템) · 어디서 도나(Claude Code 내부·독립·플랫폼) · 사용자(개발자·비개발자) · 외부 의존(OSS·물리 디바이스).

## 5. 에이전트 vs 유사 개념 비교 ★필수

**"자동화 프로세스 = 에이전트"가 아니다.** 핵심 경계는 *"절차가 고정인가, 목표를 받고 절차를 스스로 구성하는가"*.

| 개념 | 동작 방식 | 적응성 | 에이전트와의 결정적 차이 |
|------|----------|--------|------------------------|
| **스크립트** | 정적 코드를 그대로 실행 | 없음 | 환경 인지·판단 없음 |
| **RPA** | UI 기반 규칙 자동화(클릭·입력) | 규칙 벗어나면 실패 | 의미 이해·의사결정 없음 |
| **워크플로우 자동화** | 사전 정의된 흐름, 분기 로직만 조건적 | 낮음 | 흐름을 동적으로 재구성 못 함 |
| **챗봇** | 대화 응답 중심 | 대화 한정 | 멀티스텝 실행·도구 사용 제한적 |
| **단순 자동화** | 고정 규칙(If-Then) 반복 | 없음 | 목표 지향 추론 없음 |
| **AI 에이전트** | **목표 중심 — 동적 계획·도구 선택·자기 피드백·오류 복구** | 높음 | **상황에 따라 절차 자체를 재구성** |

> **한 줄 정리**: 자동화·RPA·스크립트는 *"정해진 절차를 실행"*, 에이전트는 *"목표를 받고 절차를 스스로 만든다."* 챗봇은 *"말하는 데서 끝", 에이전트는 *"말을 수단으로 일을 끝낸다."*

## 7. 자율성 레벨

| 레벨 | 내용 |
|------|------|
| **L0 수동** | 사용자가 모두 수행, AI는 참고용 |
| **L1 제안** | AI가 초안 생성, 인간이 항상 승인 후 실행 |
| **L2 제한적 자율** | 사전 정의된 범위·가드레일 안에서 자동 실행, 예외는 사람에게 에스컬레이션 |
| **L3 조건부/목표기반 자율** | 지정 범위 내 독립 실행, 중요 지점만 인간 개입 |
| **L4~5 고자율** | 목표만 주면 대부분 자율 수행 — 2026년 기준 안전상 **제한적 적용** |

> 사용자 중심 5단계 프레임: L1 Operator · L2 Collaborator · L3 Consultant · L4 Approver · L5 Observer.
> **핵심**: 가장 가치 있는 에이전트는 완전 자율이 아니라 **스펙트럼 중간(co-pilot)** — 제안하거나, 승인받아 실행하거나, 거부권 윈도우를 두고 행동.

## 8. 단일 에이전트 vs 멀티 에이전트

| | 단일 에이전트 | 멀티 에이전트(MAS) |
|---|--------------|-------------------|
| 장점 | 설계·운영 단순, 책임 소재 명확, 빠름·저비용 | 역할 분담(Planner·Executor·Critic·Auditor)·병렬·확장성 |
| 단점 | 복잡 업무·역할 분리 한계 | 비용·지연·충돌·조율 복잡성 증가 |

- **조정 방식**: 중앙 오케스트레이터(허브앤스포크) vs 에이전트 간 메시지 기반 협력(마켓·합의)
- **품질·안전**: 리뷰어·감사 에이전트를 별도로 두어 다른 에이전트의 행동·출력을 지속 검증
- **2026 트렌드**: 계층적·그래프 기반 오케스트레이션 + 동적 팀 구성. *"강한 에이전트 1 + 보조 전문 에이전트들"* 형태가 다수.

## 9. 구축·운영 고려사항 (설계·안전·검증)

- **명확한 역할·경계** — "이 에이전트가 할 수 있는 일/할 수 없는 일"을 계약처럼 정의
- **가드레일** — 권한 최소화, 허용 API·리소스 화이트리스트, 승인 워크플로우, 비용 상한, 정책 필터
- **관측·검증** — 로그·트레이스·리플레이·시뮬레이션 환경, 회귀 테스트, A/B 테스트. 결과뿐 아니라 **추론 경로(reasoning path)**의 타당성까지 검증
- **거버넌스** — 책임자 지정, 변경 관리, 안전·규제 준수(프라이버시·보안·편향)
- **운영 원칙** — 작게 시작, 중요 작업은 human-in-the-loop, "잘 되는가"보다 **"실서비스에서 안정적 반복이 되는가"**

## 10. 2026 동향과 전망

- **에이전트 플랫폼화** — 기업용 "에이전트 OS·허브" 위에 도메인 에이전트를 올리는 형태 확산
- **업무 단위 → 프로세스 단위** — 특정 업무 보조를 넘어 팀·부서 전체 프로세스를 운영하는 오토파일럿
- **신뢰·안전 중시** — 평가·시뮬레이션·감사 에이전트가 기본 구성요소로, 규제·컴플라이언스 연동 강화
- **인간-에이전트 협업 표준화** — 직무별 코파일럿이 표준 도구화, 조직 설계가 에이전트 중심으로 재편
- **기술 트렌드** — 장기 메모리·멀티모달·도구 사용 고도화, MCP(Model Context Protocol), Anthropic Claude Managed Agents(2026-04-08 베타), 에이전트 경제·상호운용성 표준
- **시장(검색 요약 인용 🟡)** — Gartner: 2026년 엔터프라이즈 앱 40%가 에이전트 탑재 / 2033년 $182.97B(CAGR 49.6%) / 도입 임원 74% 1년 내 ROI
- **남은 과제** — 환각, 보안·권한 오남용, 품질 편차
